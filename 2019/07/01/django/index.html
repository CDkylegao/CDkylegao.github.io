#6.17django

##一、前奏

  1. Web应用 / App — HTTP(s) — 应用层协议
    Hyper-Text Transfer Protocol / TCP

    HTTP - 80
    HTTPS - 443

    HTTP请求：

    请求行：GET /index.html HTTP/1.1
    请求头: 键值对
    空行：\r\n
    消息体：发给服务器的数据

    HTTP响应：

    响应行：HTTP/1.1 200 OK
    响应头：键值对
    空行：\r\n
    消息体：服务器返回的数据

    http://www.baidu.com:80/index.html
    https://www.baidu.com:443/index.html

    后端程序（服务器程序）

    Web应用开发框架 — Django / Flask / Tornado / Sanic

  2. MVC架构模式 —- 内容和显示分离 — MTV

    M：Model模型， 和MVC中M功能相同，和数据库进行交互。
    T：Template模板，和MVC中V功能相同，产生html页面。
    V：View视图，和MVC中C功能相同，接收请求，进行处理，与M和T进行交互，返回应答。

##二、在云服务器中

  • 一般不会在云服务器中创建django项目（了解）

  1. 先创建一个文件夹： mkdir django1902

  2. 进入此文件夹：cd django1902/

  3. python3 -m venv venv

  4. source venv/bin/activate

  5. 以防出问题，先更新下pip： pip install -U pip

  6. 安装django：pip install django==2.1.8

  7. 检查安装是否成功：django-admin –version

  8. 在当前目录下创建一个django项目文件（.表示当前目录）：django-admin startproject django1902 .

  9. 查看当前目录有一个manage.py文件

  10. 运行：python manage.py runserver

  11. 将端口改为80的方法：python manage.py runserver 0.0.0.0:80

    改端口之前看看80端口是否被占用，如果占用一定要解除：systemctl stop nginx

    netstat -nap | grep 80

    一般nginx会占用80端口

  12. 进入django1902/django1902：cd django1902/

    查看里面的文件：ls

    有一个settings.py文件，改里面的配置

  13. 运行：python manage.py runserver 0.0.0.0:80
