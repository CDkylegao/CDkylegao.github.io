####使用Git实施版本控制

#####1.在本地实施版本控制

  1. 在cmd命令行中创建一个文件夹(本地仓库)

    mkdir <文件夹名>

  2. 进入文件夹中

    cd <文件夹名>

  3. 实施版本控制，在此文件夹中创建版本控制的本地仓库

    git init – 把这里做成实施版本控制的仓库(里面会有一个.git的隐藏文件，相当于本地的仓库)

    ​ – 将文件夹初始化为版本控制的仓库

  4. 在此文件夹中添加东西

  5. 查看版本控制的状态

    git status – 查看状态(工作区、缓存区、仓库)

  6. 把文件纳入版本控制

    git add – 将工作区的内容放入缓存区(暂存区)

  7. 可以再用git status 命令查看文件存储在仓库的状态(未提交)

  8. 提交命令，正式放入仓库

    git commit -m ‘提交的原因’

  9. 查看提交日志

    git log

  10. 如果在文件中写了其他东西，想撤销，执行下面命令

    git checkout – - 放弃工作区修改的内容

  11. 将文件放入工作区

  12. 将所有的文件和非空文件夹纳入版本控制

    git add .

  13. 不想把以上文件提交了，返回上一个版本

    git reset –hard HEAD^

    git reset –hard - 回到指定历史的版本

  14. 可以查看未来版本

    git reflog

  15. 回退到指定的历史版本

    git reset –hard - 回退到指定的历史版本

  16. git clone – 从仓库克隆代码到本地

  17. git remote add origin – 添加远端服务器

  18. 判断是否加好

    git remote —- 有origin返回就成功了

  19. 把本地代码推到服务器仓库

    git push -u origin maste

  20. 将服务器代码下拉到本地

    git pull

#####2. 在pycharm中实施版本控制

  1. 验证有没有git

    回到主界面Configure-Settings-点Git-点Test进行测试，显示git版本号就可以了

  2. 点击Check out from Version Control-点击Git-输入在码云的仓库的URL-点击Test-测试成功就行

  3. 创建虚拟环境

    File-Settings-Project:项目名-Project Interpreter-点齿轮-点Add…-名字不修改

    venv – virtual environment – 虚拟环境

  4. 可以点击Terminal，查看环境是否成功

  5. 创建一个.gitignore文件，在里面放在版本控制的时候可以忽略的文件和文件夹

    里面写.idea 和 venv

  6. 安装pymysql依赖项，使用国内的豆瓣镜像（速度会很快）

    pip install pymysql -i https://pypi.doubanio.com/simple/

  7. 小插曲：更新pip

    python -m pip install -U pip

  8. 查看安装的依赖项以及对应版本有哪些

    pip freeze

  9. 把依赖项添加到文件里面

    pip freeze > requirements.txt

    将这个文件一定要纳入版本控制，因为将来别人来克隆这个项目，照着这个文件里面的依赖项一样一样装上就行。

  10. 将本地文件推到服务器

    点√-点VCS-Git-push

#####3.另一个程序员参与这个项目

  1. 点击Check out from Version Control-点击Git-输入在码云的仓库的URL

  2. 创建虚拟环境

  3. 照着装有依赖项的文件里的依赖项装一下

    pip install -r requirements.txt

#####4. 如果用git做项目版本控制的时候，项目已经做到里程碑版本了

  1. 用git tag命令给项目打标签，如果没有指定版本的时候，就是给当前版本打标签
  2. 发里程碑版本（打版本号）：git tag
  3. git tag v1.0 第一版原来的版本号
  4. 表示发布的第二版：git tag v2.0
  5. 查看标签：git tag
  6. 如果想让代码回退到v1.0版本：git reset –hard v1.0

#####总结

作为项目的所有者或管理者，创建好虚拟环境添加好依赖项之后一定要生成一个名为requirements.txt的文件，对应的命令是：
pip freeze > requirements.txt
将来其他开发者克隆项目以后如果需要重建项目的依赖项，可以直接使用下面的命令：
pip install -r requirements.txt
